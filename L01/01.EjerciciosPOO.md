## Configuración de Codespaces

### Objetivos

- Configurar un entorno de desarrollo profesional en la nube usando GitHub Codespaces.
- Implementar una aplicación en Java 21+ siguiendo estándares de ingeniería.

### Requisitos

1. Cuenta activa en GitHub.
2. (Recomendado) Haber solicitado el GitHub Student Developer Pack para ampliar las horas gratuitas de cómputo.

### Configuración del Entorno (The "DevContainer")

En ingeniería de software, la configuración de infraestructura como código es vital. No instalaremos Java manualmente, dejaremos que el contenedor lo haga por nosotros.

1. Crear Repositorio: Crea un nuevo repositorio privado o público llamado cloud-java-lab01.
2. Definir el Entorno: Dentro del repositorio, crea la siguiente estructura de carpetas: .devcontainer/.
3. Archivo de Configuración: Crea el archivo .devcontainer/devcontainer.json y pega el siguiente código:

```json
{
  "name": "Java Cloud Environment",
  "image": "mcr.microsoft.com/devcontainers/java:21",
  "customizations": {
    "vscode": {
      "extensions": [
        "vscjava.vscode-java-pack"
      ]
    }
  },
  "postCreateCommand": "java -version"
}
```

4. Lanzar Codespace: Haz clic en el botón Code -> pestaña Codespaces -> Create codespace on main.

### Generando el proyecto

Una vez que el IDE cargue en tu navegador, abre la terminal integrada (Ctrl + J) y sigue estos pasos:

#### Generar el proyecto

Por la terminal:

```
$ mkdir -p src/main/java/pe/edu/ulima/is2
$ touch src/main/java/pe/edu/ulima/is2/Main.java
```

#### Código de prueba

Abrir el archivo Main.java y copiar el siguiente código.

```java
package pe.edu.ulima.is2;

public class Main {
    public static void main(String[] args) {
        System.out.println("Hola IS2");
    }
}
```

### Ejecución y pruebas

En la terminal:

```
$ javac src/main/java/com/universidad/Main.java -d out
$ java -cp out pe.edu.ulima.is2.Main
```

## Ejercicios

### Ejercicio 1

Crea un sistema para gestionar una biblioteca. Debes tener clases como Libro, Autor, Usuario, Biblioteca y Prestamo. Utiliza herencia para diferentes tipos de libros (novela, ensayo, etc.) y polimorfismo para manejar diferentes tipos de usuarios (estudiante, profesor, etc.). Implementa genericidad para almacenar colecciones de libros y usuarios. Utiliza excepciones para manejar errores como libros no disponibles o usuarios que no pueden tomar prestados.

(*) Complemente con  métodos para buscar libros por título, autor o tema, prestar y devolver libros, y generar informes de préstamos.


### Ejercicio 2

Diseña un juego de rol donde los personajes tienen diferentes habilidades (ataque, defensa, magia, etc.). Utiliza herencia para crear diferentes tipos de personajes (guerrero, mago, arquero, etc.) y polimorfismo para que cada personaje tenga su propia forma de usar las habilidades. Implementa genericidad para crear listas de habilidades y excepciones para manejar errores como habilidades no disponibles o personajes sin energía

Crea un sistema de combate donde los personajes pueden atacar, defender y usar habilidades especiales.


### Ejercicio 3

Crea un sistema para gestionar las reservas de un hotel. Debes tener clases como Habitacion, Cliente, Reserva y Hotel. Utiliza herencia para diferentes tipos de habitaciones (simple, doble, suite, etc.) y polimorfismo para manejar diferentes tipos de clientes (regular, VIP, etc.). Implementa genericidad para almacenar listas de reservas y excepciones para manejar errores como habitaciones no disponibles o reservas canceladas

Implementa métodos para buscar habitaciones disponibles, realizar reservas, cancelar reservas y generar informes de ocupación..


### Ejercicio 4

Diseña una aplicación para dibujar diferentes figuras geométricas (círculos, cuadrados, triángulos, etc.). Utiliza herencia para crear las diferentes clases de figuras y polimorfismo para dibujar cada figura de forma específica. Implementa genericidad para almacenar listas de figuras y excepciones para manejar errores como parámetros inválidos.


Permite al usuario seleccionar figuras, cambiar sus propiedades (color, tamaño, posición) y guardarlas en un archivo.


### Ejercicio 5

Crea un sistema para gestionar los empleados de una empresa. Debes tener clases como Empleado, Departamento, Cargo y Nomina. Utiliza herencia para diferentes tipos de empleados (administrativo, técnico, gerente, etc.) y polimorfismo para calcular el salario de cada empleado según su cargo y departamento. Implementa genericidad para almacenar listas de empleados y excepciones para manejar errores como empleados no encontrados o salarios inválidos.

Implementa métodos para asignar empleados a departamentos, calcular salarios, generar informes de nómina y buscar empleados por nombre o cargo.
